# Repository Guidelines

## Project Structure

This repository is currently empty. Keep source code, tests, documentation, and tooling in clearly named top-level directories as the project takes shape.

## Development Notes

- Prefer small, focused changes.
- Keep generated files out of version control unless they are required by the project.
- Add or update tests alongside behavior changes once a test framework exists.
- Document new setup, build, and run commands in this file or the project README.

## Git Hygiene

- Do not rewrite existing user changes.
- Keep commits scoped to one logical change.
- Run the relevant checks before committing when project tooling is available.
